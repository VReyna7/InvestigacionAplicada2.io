LocalStrage

Es un objeto de almacenaje Web. Permite guardar una calve junto con el valor de esta en el navegador. Lo interesante de LocalStroge es que la supervivencia de los datos es hasta un reinicio completo del navegador. LocalStorage trabaja de manera similar a las cookies. Sin embargo, cuenta con un espacio de solo 5mb, y al trabajar de manera local no hace peticiones de servidor. Local Storage solo puede almacenar Strings en sus valores. Es importante saber que los datos guardados en localStorage pueden ser accedidos por cualquier script.

Sintaxis.

Agregando un dato a nuestro LocalStorage


Obteniendo un dato de nuestro LocalStorage


Removiendo un dato de nuestro LocalStorage

Ventajas


  • LocalStorage utiliza entre 5 y 10MB dependiendo del navegador web el cual estemos utilizando .
  • No existe una caducidad para localStorage, la información quedará almacenada hasta que se elimine expresamente. Aunque se cierre el navegador.

Desventajas


  • Sólo se puede almacenar cadenas de texto. O sea, no se puede guardar booleanos, tampoco guardar arrays, objetos, float. solo se guardan strings.

Ejemplo

Nuestro ejemplo se basa en la selección de un color para el fondo de una página web. Básicamente es la posibilidad Seleccionar diferentes fondos. Todo partiendo de un input color, y este almacenando su valor seleccionado en nuestro localStorage. De esa manera mantiene el color de fondo aun que se cierre el navegador.



SessionStorage

Permite acceder a un objeto storage asociado a la sesión actual. la información almacenada en esta se elimina al finalizar la sesión de la página. La sesión de la página perdura mientras el navegador se encuentra abierto, y se mantiene por sobre las recargas de reaperturas de la página.

El SessionStorage tiene una gran similitud con el localStorage lo único que los diferencia es que la diferencia entre éstas dos es que localStorage almacena la información de forma indefinida o hasta que se decida limpiar los datos del navegador y sessionStorage almacena información mientras la pestaña donde se esté utilizando siga abierta, una vez cerrada, la información se elimina.

Sintaxis

Agregando un dato a nuestro SessionStorage


Obteniendo un dato de nuestro LocalStorage


Ventajas


  • El acceso a la información es rápido, dado que los datos se almacenan de la manera llave-valor

Desventajas


  • Los valores se pierden una vez que se cierra la ventana
  • Los valores que almacena no pueden ser compartidos entre más de una ventana

Ejemplo

Nuestro ejemplos se basa en un formulario que guarda los datos en el sessionStorage, esto quiere decir que la información solo perdura mientras se mantenga la pestaña abierta. En el ejemplo se puede colocar la informacion y se mostrara al usuario, pero si abre otra ventana y ingresa a la página otra vez la información se vuelve a solicitar ya que sessionStorage solo perdura en su misma pestaña






IndexedDB

Es una manera de almacenar datos dentro del navegador del usuario. Debido a que permite la creación de aplicaciones con habilidades de consulta enriquecidas, con independencia de la disponibilidad de la red, esta es una base construida dentro del navegador, mucho más potente que localStorage. Este almacena todo tipo de valores por claves, soporta consultas de rango por clave y transacciones para confidencialidad, de igual forma este puede almacenar mucho mayor volumen de datos que localStorage

Sintaxis

Sintaxis para empezar a trabajar con indexedDB:

Let openRequesst = indexedDB.open(name, version);

Name = nombre de la base de datos

Version = entero positivo, predeterminado en 1.

En el ejemplo presentado a continuación podemos observar que es una estructura de almacenamiento en la cual muestra las diversas formas de definir claves primarias en los almacenes de los objetos.

Ventajas


  • Nos permite crear una base de datos de un almacenamiento ilimitado.
  • IndexedDB tiene un mecanismo integrado de "control de versiones de esquema", ausente en las bases de datos del lado del servidor a diferencia de otras bases de datos .

Desventajas


  • Es que no esta soportada para todos los navegadores y que son pocos los cuales toman esta base de datos dentro del navegador
